Chapter 6 :  Best Practices for Coding HTML and CFML



General Style Guidelines

This section offers some general guidelines for writing clean code. It also shows you how to use Allaire HomeSite or ColdFusion Studio to control your code style:

  • Set standards for your site
  • Reuse common code
  • Indent your code
  • Set line length and word wrapping
  • Keep capitalization consistent
  • Add comments to your code
  • Watch your URLs
  • Use special characters carefully

Set standards for your site

  • Establish a style guide for your site and stick to it. Make it available to all developers, authors, content editors, and freelancers.
  • Do not overlap tag sets:

    Wrong: <H1><A HREF="travel.html">My travel diary</H1></A>

    Right: <H1><A HREF="travel.html">My travel diary</A></H1>

  • Decide on a central location for images.
  • Create a production checklist. Every time you upload a page or deploy a whole site, run through a basic production pass: run the spell checker, validate the HTML, check your links. Naturally the more complex the application, the more QA you'll need to do.
  • Include a way for users to offer feedback and report broken links or script errors. This can be as simple as providing an email link to the Webmaster or content producer.
  • Always include descriptive page titles. Remember that text in the <TITLE> tag appears in users' bookmarks or favorites lists when they bookmark your page. It also appears in the list of titles returned in a Verity search collection or by an Internet search engine.

Reuse common code

  • Use the Code Snippets feature in Allaire HomeSite or ColdFusion Studio to store code for headers, footers, corporate logos, navigation bars, etc.
  • Use Custom Templates to enforce corporate or site-wide page styles. (Choose Save > Save As Template to save a page as a custom template.)
  • Reuse common code to reduce errors and make common page elements easier to create and update.
  • Use the Code Templates feature as a typing shortcut. This feature lets you set up abbreviations for text you type frequently.

Indent your code

  • Use white space and indent your code to set off logical code chunks.

    This is particularly useful to show the organization of nested tags that contain
    many attributes or additional tags. Use the Edit > Indent and Unindent
    commands to adjust the indentation of selected text and tags.

  • The default tab size in Allaire HomeSite and ColdFusion Studio is 4 spaces. To change it, choose Options > Settings or press F8 to open the Settings dialog box. On the Editor pane, you can adjust the Tab Width and control whether tabs are inserted as spaces.
  • On any given tag's Tag Editor dialog box (F4) you can choose to output that tag's code all on one line or indented.

Set line length and word wrapping

  • Keep line lengths below 80 characters -- at 80 characters, some editors start line wrapping.
  • Choose Options > Word Wrap or click the Word Wrap icon on the Editor toolbar to wrap long lines in the Editor pane.

Keep capitalization consistent

In HTML and CFML, it doesn't matter how you capitalize tags, attributes, and values. But it does help to choose one style and be consistent.

  • Some developers find it helpful to use uppercase for tag and attribute names, and mixed-case or lowercase for attribute values.

    <FONT FACE="Arial Black" SIZE="+2" COLOR="#FFFFFF">My Title</FONT>

  • It's a good idea to keep your web applications case-consistent. You may want to run it on another platform later. Use lowercase for filenames. Keep in mind that UNIX is case-sensitive.
  • To change the capitalization of all tags in a file, choose Edit > Convert Tag Case.

In HomeSite or Studio, you can use the Options > Settings dialog box (F8) to control the capitalization and appearance of your code in these ways:

  • Use lowercase for all inserted tags (HTML tab).

    <img src="images/logo.gif" height=1 width=1 vspace=2 border=0>

  • Force filenames to be lowercase for all <A> and <IMG> tags (HTML tab):

    <a href="directions.htm">How to Find Us</a>

  • When editing tags, return the output on a single line in the editor (HTML tab).

    <CFFORM NAME="form1" ACTION="submit.cfm" METHOD="Post">

  • Choose the color scheme used to display different HTML and CFML tags (Color Coding tab).

Add comments to your code

If the art of writing is rewriting, the art of maintaining a web site is reusing code wisely. Comments help immensely -- especially if someone other than you will update, inherit, or reuse your code.

  • Add comments to show other developers or authors how your code works or where they need to insert content. Comments also help to remind yourself how/why you did something. This makes updates and working in a team much easier.
  • Make sure the comments explain what you are trying to do, rather than stating what the code does.

    Good:
    <!--- Move to next decile --->
    <CFSET x=x+10>

    Not so good:
    <!--- Increase x --->
    <CFSET x=x+10>

  • Comments can be brief or span many lines.
  • Remember to update and sign your comments as you rewrite or add code.
  • If you're writing straight HTML, use HTML comments (<!-- -->). If you're writing CFML code, be sure to use the CFML comment tag (<!--- --->).
  • To add the HTML comment tag to the QuickBar, choose Options > Customize Toolbars, drag the Comment tool icon to the QuickBar, and click OK. The CFML Comment tag appears on the CFML Basic toolbar.
  • Keep in mind that in HTML files, comments are visible to end users who view the source of your documents. CFML comments are not visible to end-users, so these comments afford a greater degree of privacy.
Note ColdFusion comments add white space to your pages; in ColdFusion applicati, use the CFSETTING tag to prevent this.

Watch your URLs

  • Keep URLs short. Make them descriptive of site structure, where possible.
  • It's a good idea to keep your web applications case-consistent. You may want to run it on another platform later. Keep in mind that UNIX is case-sensitive.
  • You may want to use all lowercase because some users may need to type your URL exactly as configured.
  • Use the URLEncodedFormat function if you are passing dynamic parameters that may contain spaces. This prevents the server from truncating the URL at the first space. See the CFML Language Reference for more information.

Use special characters carefully

  • Be careful with special characters in web content. The web uses only a portion of available character sets. Avoid unexpected substitutions or mistakes by using character entities for special characters. For example, AT&T should be written as AT&amp;T.
  • In HomeSite or Studio, choose View > Special Characters to show the Special Characters pane, which offers a quick way to add character entities to your documents.
